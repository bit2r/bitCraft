# KTUG(noname's KTUG Shelter) 한국어 조판 자료 — 본 프로젝트 반영 검토

> 작성일: 2026-06-17
> 출처 사이트: [noname's KTUG Shelter](https://nonametex.imweb.me/)
> 검토 대상: quarto-craft 디자인 시스템(`scss/quarto-theme.scss`)의 한국어 최적화 규칙

## 결론 (두괄식)

사이트에서 **본 프로젝트에 가장 적합한 자료는 `kopunct` 패키지(2025-09-27, v1.3)** 다.
이는 국립국어원 「한글 맞춤법」 부록의 **문장 부호 조판 규약**을 코드로 정리한 것으로,
현재 quarto-craft가 다루지 않는 **유일한 공백 영역**(문장부호 조판)을 정확히 채운다.

현 프로젝트는 행간·자간·줄바꿈·폰트는 토큰화돼 있으나(`line-height: 1.75`, `letter-spacing: -0.02em`, `word-break: keep-all`),
**문장부호 선택·간격 규약은 전무**하다. kopunct는 LaTeX 매크로지만, 그 *규약 자체*는
CSS/Quarto 환경으로 옮길 수 있는 한국어 타이포그래피 표준이다.

## 2. kopunct 패키지가 정리한 한국어 문장부호 규약

> 원문: 강우현, "한국어 문장 부호를 조판하는 kopunct 패키지" v1.3 (2025-09-27).
> 배포: TeX Live `texmf-dist/doc/latex/amckorean/kopunct-doc.pdf`
> ([미러 예시](https://mirror.ischo.org/KTUG/texlive/texmf-dist/doc/latex/amckorean/kopunct-doc.pdf))

핵심 문제의식: 한글 키보드에는 문장부호 13종(마침표·물음표·느낌표·쉼표·쌍점·빗금·큰따옴표·작은따옴표·소괄호·중괄호·대괄호·붙임표·물결표)만 있어, **나머지 정식 문장부호가 입력 불편 탓에 잘못 쓰인다.**

### 2.1 규약 요약표

| 부호 | 정식 글자 | 용도 / 규칙 | 흔한 오용 |
|---|---|---|---|
| 가운뎃점 | `·` (U+00B7) | 대등·열거 항목 구분 (아침·점심·저녁) | 마침표·슬래시로 대체 |
| 3점 줄임표(한국어) | `…` (U+2026) | 말줄임 | 마침표 3개 `...` |
| 6점 줄임표 | `……` | 한국어 관행상 더 흔히 쓰임 | — |
| 영어 줄임표 | `. . .` (간격 있는 3점) | 영문 인용 | 한국어 줄임표와 혼동 |
| 홑낫표 | `「 」` | 글·작품 제목 | 작은따옴표 |
| 겹낫표 | `『 』` | 책·신문 제목 | 큰따옴표 |
| 홑화살괄호 | `〈 〉` | 작품·곡 제목 | 부등호 `< >` |
| 겹화살괄호 | `《 》` | 책·전집 제목 | — |
| 대괄호(2014 개정) | `[ ]` 기본, `〔 〕` 병용 | 묶음·보충 | — |
| 렌즈꼴 괄호 | `【 】` | 표제·강조 (【긴급】) | — |
| 겹렌즈꼴 / 겹대괄호 | `〖 〗` / `〚 〛` | 특수 표제 | — |
| 모점 | `、` | 세로쓰기/옛 문헌의 쉼표 | — |
| 고리점 | `。` | 세로쓰기/옛 문헌의 마침표 | — |
| 물결표(범위) | `~` (반각) | 범위 (제1~2장, 381~403쪽) | 수학 `∼`, 틸데 `˜` |

### 2.2 간격(spacing) 관련 핵심 통찰

디자인 시스템에 직접 시사점이 큰 두 가지:

1. **모점·고리점의 반각 처리** — 본래 전각 글자이나, 한국어 조판 시 **반각 + 뒤 공백**이 더 자연스럽다. kopunct는 글꼴 글리프가 전각으로 디자인돼 있어도 강제 반각 처리한다.
2. **물결표 모양·폭 보정** — 기본 글꼴(영문)에서는 물결표가 틸데 악센트처럼 그려져 어색하므로 확대·상하 위치(`\TildeScaleFactor=1.5` 등)를 보정한다. 즉 **범위 표기 물결표는 별도 글리프/보정이 필요**하다는 인식.

이 "전각·반각 폭 조정"은 CSS의 **`text-spacing-trim`**(CSS Text L4)과 정확히 같은 문제 영역이다.

## 3. 본 프로젝트(quarto-craft) 반영 항목

현재 `scss/quarto-theme.scss`의 한국어 규칙과 비교한 격차 및 제안.

### A. (권장·즉시) 한국어 문장부호 스타일 가이드 문서화

코드가 아닌 **저작 가이드**가 가장 비용 대비 효과가 크다.
디자인 시스템 문서(예: `images/05-korean-optimization.svg` 또는 신규 가이드 페이지)에
위 2.1 규약표를 "한국어 문서 문장부호 권장 사용법"으로 수록.

- 제목엔 `「」『』〈〉《》`, 열거엔 가운뎃점 `·`, 범위엔 물결표 `~`, 말줄임엔 `…`(필요시 `……`).
- 부등호·슬래시·마침표 연타 등 오용 패턴을 "금지 패턴"에 추가.

### B. (권장) CSS `text-spacing-trim` 도입 — 전각/반각 자동 조정

kopunct의 "반각 강제" 문제를 웹에서 표준적으로 푸는 방법.

```scss
/* scss:rules — 한국어 본문 전각 문장부호 간격 정리 */
body, .quarto-document {
  text-spacing-trim: trim-start;   /* 행두 전각 괄호/따옴표 여백 제거 */
}
```

- Chromium 123+ 지원, 점진적 향상(progressive enhancement)으로 안전하게 적용 가능.
- 미지원 브라우저는 무시 → 회귀 위험 없음.

### C. (선택) 큰따옴표/괄호 행두 걸침 — `hanging-punctuation`

```scss
.quarto-document p {
  hanging-punctuation: allow-end first;
}
```

- 현재 Safari 중심 지원. 한국어 인용·괄호 시작 단락의 좌측 정렬 안정화.
- 보조 효과이므로 디자인 토큰화까지는 불필요, 실험 항목으로만 둘 것.

### D. (선택·고급) 물결표/줄임표 입력 보조

kopunct의 "입력 불편 → 오용" 문제를 Quarto 레이어에서 완화.

- **Quarto shortcode 또는 Lua 필터**로 제목 마크업을 자동 변환:
  예) `{{< title 한글맞춤법 >}}` → `『한글맞춤법』`, `~`를 범위 문맥에서 적정 글리프로.
- 다만 과설계 위험. **A(가이드 문서)부터 적용**하고 수요 확인 후 도입 판단.

### 적용 우선순위

| 순위 | 항목 | 성격 | 비용 |
|---|---|---|---|
| 1 | A. 문장부호 가이드 문서화 | 콘텐츠 | 낮음 |
| 2 | B. `text-spacing-trim` | SCSS rules | 낮음 |
| 3 | C. `hanging-punctuation` | SCSS rules(실험) | 낮음 |
| 4 | D. shortcode/Lua 입력 보조 | Quarto 확장 | 높음 |

## 4. 반영 시 주의

- kopunct는 **LaTeX 전용 매크로**다. 코드를 그대로 가져올 수 없고, *규약(어떤 글자를 어떤 간격으로)*만 이식한다.
- B/C는 브라우저 의존 기능 → 반드시 **점진적 향상**으로, 토큰 하드코딩 금지(프로젝트 금지 패턴 준수).
- Typst PDF 출력 경로에는 `text-spacing-trim`이 적용되지 않으므로, PDF는 글꼴/조판 엔진 차원의 별도 검토 필요(현 검토 범위 밖).
